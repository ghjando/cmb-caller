3.資料傳送時機：
  -- 異動傳送："號碼有變動" 時即時傳送，"change" 欄位設為 true。
  -- 例行傳送：每分鐘回報一次 "當前號碼" 及 "叫號機離線狀態"。(為了使 "叫叫我" 顯示 "資料更新時間" 及 "離線狀態")
               ("change"欄位：call_num 或 last_update 有異動時，設為 true)
               last_update 大於10，則停止傳送，等恢復連線再繼續。
=======================================

4.Cloud Run 定時斷線說明：
"叫號資料接收器" 執行於 Cloud Run，每隔1小時(已設定為最長時間)會關閉 Socket 連線。
建議 Client程式 在連線後接近1小時的時間(例如：3500秒)，自行產生一個 新Socket 來取代 舊Socket 傳送資料。
這樣做的好處是可以無縫接續，不會有 "被斷線再重連" 的延遲時間(實測大約5秒)。
=======================================
